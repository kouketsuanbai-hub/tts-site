# CLAUDE.md

Claude Code が本リポジトリで作業する際に毎回参照する指示書。

## このプロジェクトについて
ブラウザ完結（Web Speech API）の無料テキスト読み上げサイト。  
詳細仕様は `tts-site-spec.md` を参照。**先にそちらを読むこと**。

## 絶対に守ること
1. **完全無料の前提を崩さない**  
   クラウド TTS API、課金が発生する外部サービス、バックエンドサーバー、有料の依存パッケージを追加しない。
2. **依存ゼロ**  
   npm / yarn / pnpm を使わない。React / Vue / フレームワーク類を導入しない。バニラ HTML/CSS/JS のみ。
3. **テキストは外部に送信しない**  
   入力テキストを外部 API や解析ツールに送る処理を追加しない。プライバシーがこのサイトの売りである。
4. **仕様書からの逸脱は事前確認**  
   `tts-site-spec.md` に書かれていない判断は勝手に進めず、纐纈に確認する。

## コーディング規約
- HTML: `<!DOCTYPE html>` + `lang="ja"`
- CSS: 色は `:root` のカスタムプロパティで管理
- JS: IIFE で名前空間を汚さない
- インデント: スペース2
- コメントは必要な箇所のみ、日本語で簡潔に

## Web Speech API の注意点（必ず実装に反映）
- 1発話が15秒を超えると Chrome で停止する → 文単位で分割して連結再生
- 8秒ごとに pause→resume で keepalive を入れる
- `getVoices()` は `onvoiceschanged` で再取得
- セッションIDで停止時の連結中断を確実にする
- Safari は `onboundary` 未発火の可能性あり、ハイライトは best-effort

## コミットルール
- 1フェーズ完了ごとにコミット
- メッセージは日本語で `feat: 〇〇を追加` / `fix: 〇〇を修正` / `docs: 〇〇を更新` の形式
- まとめてコミットしない。論理単位で分ける。

## 作業の進め方
1. 仕様書（`tts-site-spec.md`）の Phase を確認
2. 現在の Phase の完了条件を一つずつ消化
3. ブラウザで動作確認
4. コミット
5. 次の Phase に進む前に纐纈に進捗を報告

## 動作確認方法
- ローカル: `python -m http.server 8000` または Live Server 拡張
- Node.js は不要

## 不明点の扱い
仕様書にない判断、外部サービスの利用、データ保存の仕組み追加は、勝手に決めずに必ず確認する。
