## 結論サマリー

- `[事実]` 「完全無料・登録不要・文字数無制限・送信なし・極端にシンプル」を“すべて”満たすサービスは、現時点で確認できる範囲では特定できなかった。最も近いのは ao-system.net「テキスト読み上げ」（Web Speech API・無料・登録不要・送信なし）だが、無料系サイトの多くは文字数制限・クレジット表記・登録のいずれかで引っかかる。
- `[仮説（裏未取得）]` 「ChatGPT の長文をコピペ→端末内蔵音声で読む／一切登録なし／文字数無制限／極シンプル」という5条件“同時”のポジションは、相対的に空いている可能性が高い。ただし「個別の条件を満たすサイト」は多数あり、差別化の核は“5条件の同時成立＋UI の極端な単純さ”に絞られる。
- `[事実]` 高精度サービス（音読さん・CoeFont・ElevenLabs・クラウド TTS）は軒並みクラウド送信型で、無料枠に明確な文字数上限がある。「送信なし・無制限」は技術的に Web Speech API（端末内蔵音声）でしか成立しにくく、ここが構造的な差別化点。
- `[仮説（裏未取得）]` 広告収益化（AdSense）は、単機能ツールサイトが「有用性の低いコンテンツ」「最小要件未達」で審査落ちしやすいという一般論があり、ツール単体では通りにくい懸念。SEO 競合（「テキスト 読み上げ 無料」）は比較まとめ記事・大手ツールがひしめき、厚い。

## 2. 「完全無料・登録不要・無制限・送信なし・シンプル」を全部満たすか

- `[事実]` 上記いずれも、5条件“全て”同時には満たさない。
  - ao-system：送信なし・無料・登録不要は満たすが、文字数無制限は明記なし（長文版を分けている＝標準版に制限の存在を示唆）。
  - 音読さん／TTSMaker／THE NARRATOR：文字数制限あり（送信もクラウド）。
  - VOICEVOX：無制限・送信なしだが「ソフトの DL/インストールが必要＝ブラウザ完結でない・極端なシンプルさに反する」。
- `[仮説（裏未取得）]` 「ブラウザ完結＋Web Speech API＋無制限＋登録不要＋送信なし＋極シンプル UI」を同時に名乗るサービスは、今回の調査範囲では特定できなかった。技術的には Web Speech API を使えば「無制限・送信なし」は容易に実現でき、参入障壁が低い＝同型の小サイトが他にも存在する蓋然性は高い（裏取り経路：「テキスト 読み上げ Web Speech API サイト」での個別サイト悉皆調査、GitHub Pages 上の同種実装の棚卸し）。このポジションは「空いている」と断定はできないが、明確な看板を掲げた既存の強い競合は見当たらず、相対的に手薄。

## 3. 高精度・有料寄り代表例と無料枠の文字数

- `[事実]` 音読さん：無料 1,000字/月（非会員）・5,000字/月（会員）。有料 980円〜2,980円/月で 20万〜100万字。
- `[事実]` CoeFont：無料は3音声・営利利用不可。Standard はポイント制（40万pt/月・1字 5pt 目安）。
- `[事実]` ElevenLabs：無料 10,000 クレジット/月（約1万字・高品質モデルで約10分）。無料は商用不可・要クレジット表記。
- `[事実]` Amazon Polly：標準音声 500万字/月が無料だが「最初の12か月のみ」。
- `[事実]` Google Cloud TTS：標準音声 400万字/月・WaveNet 100万字/月が無料、こちらは恒久（月次更新）。
- `[事実]` VOICEVOX：商用・非商用とも無料、要クレジット表記。ローカル実行で文字数制限なし。

## 4. Web Speech API（ブラウザ内蔵）ベースの無料サイトの存在量

- `[事実]` Web Speech API はブラウザ標準・無料で、SpeechSynthesis を数行の JS で実装できる（MDN・各技術ブログ多数）。実装ハードルが極めて低く、Qiita・個人技術ブログ等に「コピペで動く」実装記事が多数ある＝同型サイトを誰でも作れる状態。
- `[事実]` 稼働中の実サービスとしては ao-system.net「テキスト読み上げ」が Web Speech API 明記の代表例。kaihatuiinkai.jp は「サンプル/デモ」寄り。
- `[仮説（裏未取得）]` 「テキスト 読み上げ 無料」上位は、(a) AI 音声クラウド型（音読さん・TTSMaker・各まとめ記事）と (b) 比較・ランキングまとめ記事が主流で、純粋な Web Speech API 単機能サイトは SEO 上位を取れていない傾向に見える。つまり「実装は氾濫しているが、検索で目立つ位置にいるのはクラウド AI 音声側」という構図（裏取り経路：実際の検索結果順位の定点観測）。

出典：[MDN Web Speech API](https://developer.mozilla.org/ja/docs/Web/API/Web_Speech_API)／[Twilio解説](https://www.twilio.com/ja-jp/blog/speech-to-text-browser-web-speech-api-jp)／[ao-system.net](https://ao-system.net/speak/)

## 5. 広告収益化の現実性メモ

- `[仮説（裏未取得）]` AdSense 審査では「有用性の低いコンテンツ」「最小要件（コンテンツ量・独自性）未達」が頻出の不合格理由とされ、記事のない単機能ツールサイトは通りにくいという一般論が複数の解説で示されている。ツール単体ページのみでの合格は不利で、使い方・FAQ・読み物（コラム）等のコンテンツを併設して通すのが定石、というのが二次情報の共通見解（一次＝Google の公式審査基準原文での「ツールサイト不可」の明文は未確認＝`[未確認]`）。
- `[仮説（裏未取得）]` SEO 競合は厚い。「テキスト 読み上げ 無料」は音読さん等の確立サービス＋比較まとめ記事（CyberLink・aismiley・アスピック等の SaaS メディア）が上位を占有。新規単機能サイトが純粋なツール検索で上位を取るのは難易度が高い。勝ち筋は「ChatGPT 長文の読み上げ」という用途特化のロングテール語と、5条件同時成立という機能差別化の組み合わせ、という見立て（裏取り経路：実検索順位・キーワードプランナーでの検索ボリューム調査）。

出典：[AdSense審査・有用性の低いコンテンツ（seory）](https://seory.co.jp/google-adsense-audit/)／[Xserver: 審査に通らない理由](https://www.xserver.ne.jp/blog/google-adsense-reexamination/)／[まとめ記事例（CyberLink）](https://jp.cyberlink.com/blog/audioeditor/2932/best-audio-editing-tool-for-text-to-speech)・[aismiley](https://aismiley.co.jp/ai_news/voice-reading-software-language/)
