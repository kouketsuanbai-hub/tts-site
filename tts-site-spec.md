> このドキュメントは Cursor 内の Claude Code でサイト実装を進めるための指示書である。
> 上から順にフェーズを実行すること。各フェーズの完了条件を満たしてから次に進む。

## 0. プロジェクト概要

### 0.1 何を作るか
ブラウザだけで動作する、完全無料・登録不要のテキスト読み上げサイトを公開する。  
ユーザーがテキストを貼り付け／入力し、「読み上げ」ボタンを押すと、ブラウザ内蔵の音声エンジンで読み上げる。

### 0.2 なぜ作るか（ポジショニング）
既存の無料 TTS サイトには次のいずれかの欠点がある。
- 文字数制限がある
- 長文で途中停止する
- UI が煩雑で、貼って押すまでの動線が長い
- ログイン必須・トークン制限あり
- テキストをサーバーに送るため、機密文書では使えない

このサイトは「**完全無料・登録不要・送信なし・長文でも止まらない・極端にシンプル**」のポジションを取る。  
高品質クラウド音声は採用しない（コスト発生を避けるため）。音声品質はブラウザ内蔵音声に依存することを前提とする。

### 0.3 収益化の方針
v1 では収益化機能を実装しない。まず利用実績と SEO 評価を作ることを優先する。  
v2 以降で広告（AdSense / 自社枠）またはアフィリエイトの導入を検討する。

### 0.4 採用しない選択
- クラウド TTS API（OpenAI / Gemini / Azure 等）の利用 → コスト発生のため除外
- 音声ファイル（mp3）のダウンロード機能 → Web Speech API では実装不可
- ユーザー登録・ログイン → 不要
- バックエンドサーバー → 不要（静的サイトのみ）

## 1. 技術スタック

### 1.1 確定事項
- 言語: HTML / CSS / JavaScript（バニラ）
- フレームワーク: なし（依存ゼロで保守容易性を優先）
- ビルドツール: なし（静的 HTML を直接配信）
- 音声合成: Web Speech API（SpeechSynthesis）
- ホスティング: Cloudflare Pages（無料・独自ドメイン対応・CI/CD不要）
- リポジトリ: GitHub（`kouketsuanbai-hub` 配下に新規作成）

### 1.2 選定理由
- 依存ゼロ → npm install 不要、ビルド不要、即デプロイ可能
- Cloudflare Pages → ドラッグ&ドロップ／Git連携の両対応、無料枠で十分
- Web Speech API → 追加コストゼロ、ユーザーのテキストを外部送信しない

## 3. 機能要件

### 3.1 v1 必須機能（最初のリリース）
| ID | 機能 | 詳細 |
|----|------|------|
| F-01 | テキスト入力 | 大きめのテキストエリアに貼り付け／直接入力 |
| F-02 | 読み上げ実行 | ボタン1つで開始。文字数制限なし |
| F-03 | 長文の安定再生 | 文（。！？・改行）単位で分割し、順次連結再生 |
| F-04 | 途中停止の回避 | 8秒ごとに pause→resume を呼ぶ keepalive |
| F-05 | 一時停止 / 再開 | 同一ボタンで切り替え |
| F-06 | 停止 | 再生中の発話を完全に中断 |
| F-07 | クリア | テキストエリアを空にする |
| F-08 | 音声選択 | 端末で利用可能な音声から選択（日本語を先頭に並べる） |
| F-09 | 速度調整 | 0.5x ～ 2.0x、0.1刻み |
| F-10 | 高さ調整 | 0 ～ 2、0.1刻み |
| F-11 | 文字数表示 | 入力中の文字数をリアルタイム表示 |
| F-12 | 読み上げ位置ハイライト | onboundary イベント対応ブラウザで該当箇所を強調 |
| F-13 | 選択位置からの読み上げ | カーソル位置から末尾まで／範囲選択時はその範囲だけ読み上げる。停止時は最後に読んだ位置にカーソルを戻し、続きから再生できるようにする |

### 3.2 v1 で実装しないもの
- 音声ファイル保存（mp3エクスポート）
- 設定の永続化（localStorage 等）
- ダークモード
- 多言語UI（日本語のみ）
- 利用統計
- 広告

### 3.3 v2 以降の候補
- 設定の保存（localStorage への速度・音声記憶）
- ダークモード切り替え
- URL クエリでテキスト受け渡し
- ブックマークレット
- AdSense 導入

## 4. 非機能要件

### 4.1 性能
- ページ初回表示: 1秒以内（依存ゼロのため達成容易）
- 100,000 文字の入力でもブラウザがフリーズしない
- 再生開始までの遅延: ボタン押下から1秒以内

### 4.2 ブラウザ対応
- Chrome 最新版（メインターゲット、boundary イベント対応）
- Edge 最新版（同上）
- Safari 最新版（ハイライトは動かない可能性あり、再生は動作）
- Firefox 最新版
- iOS Safari / Android Chrome（モバイル対応）

### 4.3 プライバシー
- テキストはサーバーに一切送信しない
- Cookie は使用しない（v1 時点）
- Google Analytics 等の計測タグは v1 では入れない（最小構成を優先）

### 4.4 アクセシビリティ
- キーボード操作可能（Tab フォーカス、Enter で実行）
- フォーカスリングを残す
- ボタンに aria-label を付与
- 色のコントラスト比 AA 以上

## 6. 実装の重要ポイント

### 6.1 Web Speech API の既知の罠（必ず対応すること）
1. **Chrome の約15秒タイムアウト**  
   1回の SpeechSynthesisUtterance が15秒程度を超えると、無音状態で停止する。  
   → 対策: 文単位（最大140文字程度）に分割して、`onend` で次のチャンクを連結再生する。

2. **長文（約4000文字超）でのバッファ上限**  
   1発話に長すぎるテキストを渡すと、途中で切れる。  
   → 対策: 上記の分割で同時に解決される。

3. **タブが非アクティブ／長時間再生で停止**  
   → 対策: 発話中に `setInterval` で `pause()` → `resume()` を8秒ごとに呼んでタイマーをリセットする（keepalive）。

4. **`getVoices()` が初回呼び出しで空配列を返す**  
   → 対策: `synth.onvoiceschanged` イベントで再取得する。

5. **Safari は `onboundary` を発火しない場合がある**  
   → ハイライト機能は best-effort 扱い。再生自体は必ず動かす。

### 6.2 セッション管理
停止ボタンを押した後、前のセッションのコールバックが発火して次のチャンクを再生してしまう不具合を防ぐため、セッションIDで進行中の連結を打ち切る仕組みを必ず実装する（既存プロトタイプの実装を参照）。

### 6.3 入力サニタイズ
HTML を貼り付けられても安全に表示するため、テキストエリアの内容を innerHTML で扱わない。ハイライト表示用の重ね描画では `&`, `<`, `>` をエスケープする。

### 6.4 文字数の上限
v1 ではUI上の上限を設けない。ただし極端な入力（数MB単位）に備え、入力欄の高さは固定 + スクロール可能とする。

## 8. Claude Code への作業指示

### 8.1 基本ルール
- このファイルの仕様から逸脱しない。仕様変更は必ず人間（纐纈）に確認すること。
- 1フェーズずつ完了させ、各フェーズ末でコミットする。コミットメッセージは日本語で `feat: 〇〇を追加` の形式。
- 依存パッケージを追加しない。npm / yarn / pnpm を使わない。
- バックエンドや API を勝手に追加しない（完全無料の前提を維持するため）。

### 8.2 コード品質
- HTML は <!DOCTYPE html> から始めて lang="ja" を指定
- CSS はカスタムプロパティで色を管理
- JS は IIFE またはモジュールで名前空間を汚さない
- コメントは必要な箇所のみ、簡潔に日本語で

### 8.3 動作確認
ローカル動作確認は VS Code / Cursor の Live Server 拡張、または `python -m http.server 8000` を使う。Node.js は不要。

### 8.4 不明点の扱い
仕様に書かれていない判断が必要になった場合、勝手に決めずに纐纈に確認する。特に以下は確認必須:
- デザインの大きな変更
- 機能の追加・削除
- 外部サービスの利用
- ユーザーデータを保存する仕組みの導入
